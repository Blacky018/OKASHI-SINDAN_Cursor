# お菓子診断アプリ 要件定義書

## 1. プロジェクト概要

### 1.1 目的
ユーザーが現在の気分や状況に応じて、最適なお菓子を診断・提案するアプリケーション

### 1.2 ターゲットユーザー
- お菓子が好きな人
- 何を食べたいか迷っている人
- 新しいお菓子を発見したい人

## 2. 機能要件

### 2.1 必須機能

#### 2.1.1 診断機能
- **質問形式の診断**
  - 複数の質問（3-5問程度）に回答
  - 質問例：
    - 今の気分は？（リラックス/元気になりたい/甘いものが欲しい など）
    - 時間帯は？（朝/昼/夜/深夜）
    - 予算は？（100円以下/200円以下/300円以下/こだわらない）
    - 好みの食感は？（サクサク/ふわふわ/しっとり/カリカリ）
    - 甘さの好みは？（甘め/控えめ/苦め）

- **診断ロジック**
  - 回答の組み合わせに基づいてお菓子を提案
  - 複数の候補から1つを選出（または複数提案）

#### 2.1.2 結果表示機能
- 診断結果のお菓子名
- お菓子の説明・特徴
- おすすめポイント
- 画像（可能であれば）

#### 2.1.3 再診断機能
- 診断結果から再度診断を開始できる

### 2.2 拡張機能（将来実装）

- お菓子データベースの拡充
- 診断履歴の保存
- お気に入り機能
- シェア機能（SNS連携）
- お菓子の詳細情報（カロリー、原材料など）
- カテゴリ別検索（チョコレート、クッキー、キャンディなど）

## 3. 非機能要件

### 3.1 パフォーマンス
- 診断結果の表示は3秒以内
- スムーズな画面遷移

### 3.2 ユーザビリティ
- 直感的な操作
- 分かりやすい質問文
- 視覚的に魅力的なデザイン

### 3.3 互換性
- 主要ブラウザで動作（Chrome, Safari, Firefox, Edge）
- レスポンシブデザイン（スマートフォン対応）

## 4. 技術要件

### 4.1 フロントエンド
- HTML5, CSS3, JavaScript
- フレームワーク：React / Vue.js / Next.js（推奨：Next.js）
- UIライブラリ：Tailwind CSS / Material-UI / Chakra UI

### 4.2 バックエンド（必要に応じて）
- 静的サイトとして実装可能（診断ロジックはフロントエンドで完結）
- 将来的にデータベースが必要な場合：Node.js + Express / Python + Flask

### 4.3 データ管理
- 初期はJSONファイルでお菓子データを管理
- 将来的にデータベース（PostgreSQL / MongoDB）に移行

## 5. データ構造

### 5.1 お菓子データ
```json
{
  "id": 1,
  "name": "チョコレート",
  "category": "チョコレート",
  "description": "濃厚な味わいのチョコレート",
  "price_range": "100-200円",
  "texture": ["しっとり"],
  "sweetness": "甘め",
  "mood": ["リラックス", "甘いものが欲しい"],
  "time": ["昼", "夜"],
  "image": "chocolate.jpg"
}
```

### 5.2 診断ロジック
- 質問の回答をスコア化
- お菓子データとマッチング
- 最も適合度の高いお菓子を提案

## 6. UI/UX要件

### 6.1 画面構成
1. **トップページ**
   - アプリのタイトル
   - 「診断を始める」ボタン

2. **診断画面**
   - 質問文
   - 選択肢（ボタン形式）
   - 進捗表示（何問目/全何問）

3. **結果画面**
   - 診断結果のお菓子
   - 説明文
   - 「もう一度診断する」ボタン
   - 「トップに戻る」ボタン

### 6.2 デザイン要件
- 明るく親しみやすい色合い
- お菓子の画像を効果的に使用
- アニメーション効果（フェードインなど）

## 7. 開発スケジュール（目安）

### Phase 2: UI/UX改善（1週間）

### Phase 3: テスト・調整（1週間）
